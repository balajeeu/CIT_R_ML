R apply functions

Functions covered:

  • row and column functions: rowSums, colSums, rowMeans, colMeans
  • apply family: apply, lapply, sapply, vapply, mapply, rapply, and tapply

apply

  • takes as input: matrix or dataframe
  • applies functions to margins of the matrix of dataframe (margin = row or column; margin 1 refers to rows and margin 2 refers to columns)
  • Applying function to each column (column = margin 2):
  • find max of each row in matrix_1
apply(matrix_1, 1, max) #find max of each row in matrix_1
## [1] 7 8 9
  • Applying function to each column (column = margin 2):
  • find min of each column in matrix_1
apply(matrix_1, 2, min) 
## [1] 1 4 7

lapply

  • takes as input: list or vector
  • applies function to each element of the list or vector
  • output: returns a list
  • ‘l’ because it always returns a list
#create character vector with city names
cities <- c("chennai", "new delhi", "mumbai")
#for each element in X, apply function nchar() to count number of characters
lapply(X = cities, FUN = nchar)
## [[1]]
## [1] 7
## 
## [[2]]
## [1] 9
## 
## [[3]]
## [1] 6
  • To return values as vector, use unlist:
unlist(lapply(X = cities, FUN = nchar))
## [1] 7 9 6

sapply

  • very similar to lapply, but tries to simplify the output to a vector or matrix
  • takes as input: list or vector
  • applies function to each element of the list or vector
  • output: always tries to return a vector or matrix; but if not possible, will return a list
  • ‘s’ because it always tries to simplify the output by returning a vector or matrix if possible

minAndMax <- function(x) { #create a  function that returns the min and max

vapply

  • exactly the same as sapply in terms of functionality
  • use to squeeze some more speed out of your code by telling R what sort of stuff your function will return
cities <- c("New Delhi", "chennai", "mumbai")
vapply(cities, nchar, numeric(1)) #tells vapply we're expecting 1 number for each element
## New Delhi   chennai    mumbai 
##         9         7         6

If you specify numeric(2), R will return an error message.

dat <- list(a = 1:10, b = 11:20)
vapply(dat, function(x) {return(c(min = min(x), max = max(x),mean = mean(x)))}, c(min = 0, max = 0, mean = 0)) 
##         a    b
## min   1.0 11.0
## max  10.0 20.0
## mean  5.5 15.5
#returns minimum, maximum and mean

mapply

  • multivariate version of sapply
  • multivariate in the sense that your function must accept multiple arguments
  • applies function (the first argument supplied to mapply) to each element
  • tries to return a vector
mapply(sum, 1:5, 1:5, 1:5)
## [1]  3  6  9 12 15

Another example:

mapply(rep, 1:4, 4:1)
## [[1]]
## [1] 1 1 1 1
## 
## [[2]]
## [1] 2 2 2
## 
## [[3]]
## [1] 3 3
## 
## [[4]]
## [1] 4

rapply

  • apply a function to each element of a nested list structure, recursively
  • relatively uncommon
  • r in rapply because recursive
dat <- list(a = list("A", "B", "C"), b = c(1, 100), c = list("Hey"))
dat
## $a
## $a[[1]]
## [1] "A"
## 
## $a[[2]]
## [1] "B"
## 
## $a[[3]]
## [1] "C"
## 
## 
## $b
## [1]   1 100
## 
## $c
## $c[[1]]
## [1] "Hey"
  • create custom function to supply to rapply
myFunction <- function(x) {
    if (is.character(x)) { #if element within the list is a character, add !
        return(paste0(x, "!"))
    }
    else {
        return(x + 1000) #if element isn't a character, add 1000 to it
    }
}
rapply(dat, myFunction)
##     a1     a2     a3     b1     b2      c 
##   "A!"   "B!"   "C!" "1001" "1100" "Hey!"

tapply

  • takes as input: vector
  • tells R how to group the element in the vectors
  • applies function to each group of variable
x <- c(1, 2, 3, 10, 20, 30, 100, 200, 300) #create vector
groups <- c("a", "a", "a", "b", "b", "b", "c", "c", "c") #create grouping variable (3 groups)
tapply(x, groups, mean)
##   a   b   c 
##   2  20 200

In essence, tapply has done this:

c(a = mean(c(1, 2, 3)), #mean of group a
  b = mean(c(10, 20, 30)), #mean of group b
  c = mean(c(100, 200, 300)) #mean of group c
  )
##   a   b   c 
##   2  20 200
